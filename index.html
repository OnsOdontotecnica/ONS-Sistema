SQL para Configurar RLS (Copie e execute no Supabase)

-- ============================================
-- CONFIGURAÇÃO DE SEGURANÇA - ROW LEVEL SECURITY
-- Execute este código no SQL Editor do Supabase
-- ============================================

-- 1. Habilitar RLS na tabela chamados
ALTER TABLE chamados ENABLE ROW LEVEL SECURITY;

-- 2. Política para SELECT (visualizar dados)
CREATE POLICY "Allow select for all users" ON chamados
    FOR SELECT USING (true);

-- 3. Política para INSERT (inserir dados)
CREATE POLICY "Allow insert for all users" ON chamados
    FOR INSERT WITH CHECK (true);

-- 4. Política para UPDATE (atualizar dados)
CREATE POLICY "Allow update for all users" ON chamados
    FOR UPDATE USING (true);

-- 5. Política para DELETE (excluir dados)
CREATE POLICY "Allow delete for all users" ON chamados
    FOR DELETE USING (true);

-- 6. (Opcional) Verificar políticas existentes
SELECT * FROM pg_policies WHERE tablename = 'chamados';

Passo a Passo para Configurar

  1. 1. Acesse seu painel do Supabase em https://supabase.com
  2. 2. Selecione seu projeto
  3. 3. Vá em SQL Editor no menu lateral
  4. 4. Clique em New Query
  5. 5. Cole o SQL acima no editor
  6. 6. Clique em RUN para executar
  7. 7. Pronto! Seu banco de dados está seguro com RLS ativado

O que é RLS?

Row Level Security (RLS) é um recurso de segurança que protege seu banco de dados, permitindo que apenas usuários autorizados acessem os dados. Mesmo que alguém tenha sua chave API, não conseguirá acessar ou modificar dados sem as permissões corretas.

✅ Com RLS ativado, suas credenciais ficam protegidas!
